# Label Studio MCP — Troubleshooting

Часті проблеми та їх вирішення. Базове встановлення — у
[user-guide.md](./user-guide.md).

---

## Де подивитись логи (Windows)

Логи розширення Claude Desktop:

```
%APPDATA%\Claude\logs\
```

Шукай рядки з префіксом `[Label Studio]`. Сервер пише діагностику в **stderr**,
тож повідомлення на кшталт `Connected to Label Studio at …` або помилки видно
саме там.

---

## «Unable to connect to extension server»

**Симптом:** одразу після встановлення/ввімкнення.

**Причина:** перший запуск нового бандла створює віртуальне середовище й
завантажує залежності (numpy/pandas/lxml тощо) — це разово ~15–30 с, що може
перевищити таймаут підключення Claude Desktop. Сам сервер при цьому робочий.

**Рішення:**
1. **Disable → Enable** розширення (або зачекай ~30 с і повтори). Перший запуск
   уже підготував середовище — друга спроба під'єднається за ~2 с.
2. Якщо повторюється щоразу — перевір, що `uv` доступний (див. нижче) і що є
   мережа до інтернету для першого завантаження залежностей.

## «Failed to spawn: mcp-label-studio» / «uv: program not found»

**Причина:** у PATH немає `uv` (рантайм, через який запускається сервер).

**Рішення:** встанови uv та перезапусти Claude Desktop:

```powershell
winget install astral-sh.uv
# або:
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

Перевір: `uv --version` у новому терміналі.

## Помилки TLS / сертифіката (внутрішній HTTPS)

**Симптом:** помилка під'єднання до `https://…internal.example`, згадки про
`certificate verify failed` / `SSL`.

**Причина:** HTTPS-сертифікат виданий внутрішнім (AD) CA, якому середовище не
довіряє.

**Рішення (за пріоритетом):**
1. **Доменна машина:** корінь `internal.example` зазвичай уже в сховищі Windows —
   розширення довіряє сховищу ОС автоматично, нічого робити не треба.
2. **CA Certificate:** у налаштуваннях розширення вкажи PEM-файл внутрішнього CA
   (поле «CA Certificate»).
3. **Тимчасово:** вимкни «Verify TLS certificate» (лише для довіреної внутрішньої
   мережі).

> Plain **HTTP** (`http://…`) TLS не потребує — переконайся лише, що хост
> резолвиться (DNS / членство в домені).

## Створення коментаря → 404

**Симптом:** інструмент коментарів повертає `{"error": true, "status_code": 404,
…}`.

**Причина:** API коментарів недоступне на цьому інстансі/редакції Label Studio
(коментарі тяжіють до Enterprise). Це не проблема даних чи прав.

**Рішення:** на Community-інстансі коментарі через API можуть бути недоступні —
використовуй інші інструменти; перевір потребу саме коментарів.

## Зсунуті NER-спани / помилки валідації спанів

**Симптом:** помилка `span mismatch …`, `missing the required 'text'`,
`span out of range`, `to_name … not bound`, `unknown control`.

**Це очікувано** — захист від хибної розмітки. Для спанів (`type="labels"`):
- `start`/`end` — це **символьні** індекси (0-based, end-exclusive), не байти;
- обов'язково передавати `value["text"]` = точна підрядка `data[поле][start:end]`;
- поле визначається з конфігу проєкту за `from_name`/`to_name`.

Якщо Claude отримав таку помилку — він має перерахувати офсети програмно
(`str.find` + довжина) і повторити. Якщо помилка «field … not in task data» або
«unknown control» — перевір, що `from_name`/`to_name` відповідають labeling-
конфігу проєкту.

## Після оновлення зникли налаштування

**Причина:** змінився ідентифікатор розширення (`local.mcpb.<author>.<name>`) —
тобто змінилось поле `author` або `name` у маніфесті. Тоді Claude Desktop вважає
це новим розширенням.

**Рішення:** за стабільних `author`/`name` (наш випадок — `DL-Solution` /
`label-studio-mcp`) оновлення зберігають налаштування. Якщо колись зникли —
просто введи URL/ключ ще раз; надалі буде зберігатись.

---

## Розширення «enabled», але інструменти не працюють

1. Перевір логи (вище) на помилку ініціалізації.
2. Перевір **API Key** і **URL** у налаштуваннях.
3. Перевір мережу/VPN до `internal.example`.
4. Disable → Enable; за потреби перевстанови свіжий `.mcpb`.
