# Label Studio MCP — Гайд користувача

Документ для співробітників: як встановити та користуватися розширенням Label
Studio у Claude Desktop. Технічні деталі деплою — в [README](../README.md),
вирішення проблем — у [troubleshooting.md](./troubleshooting.md).

---

## 1. Що це

Розширення (MCP-сервер) для Claude Desktop, яке дає Claude доступ до вашого
Label Studio: проєкти, задачі, анотації, предикшени, користувачі, воркспейси,
вебхуки, ML-бекенди. Після встановлення можна давати Claude запити природною
мовою, напр. «створи проєкт», «скільки задач розмічено», «додай предикшени».

## 2. Передумови

- **Claude Desktop** (актуальна версія).
- **Доступ до Label Studio** у мережі — для внутрішнього інстансу
  `https://label-studio.dk.internal.example` потрібно бути в корпоративній мережі
  або через VPN.
- **API-ключ** Label Studio (див. крок 4).
- **uv** — зазвичай надається Claude Desktop автоматично. Якщо у логах буде
  `uv: program not found` — встанови uv (див. troubleshooting).

## 3. Встановлення

1. Візьми файл `label-studio-mcp-<версія>.mcpb` (роздається всередині команди —
   напр. зі спільної теки / GitHub Release форку).
2. Відкрий **Claude Desktop → Settings → Extensions**.
3. Перетягни `.mcpb` у вікно (або відкрий його).
4. З'явиться картка **Label Studio** (Developed by DL-Solution) → натисни
   **Install**.

> Попередження «Installing will grant this extension access…» — нормальне для
> будь-якого локального розширення. Встановлюй лише бандли з довіреного джерела
> (наша команда).

## 4. Де взяти API-ключ

1. Залогінься в Label Studio (`https://label-studio.dk.internal.example`).
2. Праворуч угорі → **Account & Settings**.
3. Розділ **Access Token** → скопіюй токен.

Ключ прив'язаний до твого користувача й прав — Claude діятиме від твого імені.

## 5. Налаштування розширення

Після Install у картці розширення заповни:

| Поле | Значення |
|---|---|
| **Label Studio URL** | `https://label-studio.dk.internal.example` (або свій інстанс) |
| **API Key** | токен з кроку 4 (маскується, зберігається в сховищі ОС) |
| **CA Certificate** *(опц.)* | PEM внутрішнього CA, якщо HTTPS-сертифікат від AD і не довіряється автоматично |
| **Verify TLS certificate** | лиши увімкненим; вимикай лише якщо сертифікат самопідписаний і немає CA |

Натисни **Save**, потім увімкни перемикач **Enabled**.

> **TLS для внутрішнього HTTPS:** на доменній машині кореневий сертифікат
> `internal.example` зазвичай уже в сховищі Windows — тоді нічого додатково не
> треба. Якщо ні — або вкажи **CA Certificate** (PEM), або тимчасово вимкни
> **Verify TLS certificate**.

## 6. Перший запуск

Перше ввімкнення нового розширення може зайняти **20–40 секунд** — Claude Desktop
підтягує залежності. Якщо вискочило **«Unable to connect to extension server»** —
це таймаут першого старту, не помилка: **вимкни і знову ввімкни** розширення
(disable → enable), і воно під'єднається. Далі старти миттєві.

## 7. Оновлення

Коли команда випускає новий `.mcpb`:
1. Просто **встанови новий файл поверх** старого розширення.
2. Налаштування (URL/ключ) **зберігаються** — повторно вводити не треба.

(Автооновлення для приватних бандлів немає — встановлення ініціюється вручну.)

## 8. Базове використання

Приклади запитів до Claude (інструмент викликається автоматично):

- «Покажи список проєктів у Label Studio.»
- «Скільки задач у проєкті 12 і скільки з них розмічено?»
- «Створи проєкт "Відгуки" з міткою sentiment (Positive/Negative/Neutral).»
- «Імпортуй задачі з файлу C:\\data\\tasks.json у проєкт 12.»
- «Додай предикшен NER для задачі 31: сутність PRODUCT на "Winston Blue".»
- «Експортуй анотації проєкту 12.»

> Для NER/спанів Claude має передавати точний текст сутності та символьні
> офсети — інструмент перевіряє їх проти тексту задачі й відхилить зсунуті
> спани (захист від хибної розмітки).
